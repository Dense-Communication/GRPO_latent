## Slide 4: LatentMAS 框架介绍

**演讲要点**:
> "LatentMAS 框架提出了一种新思路：不传递原始文本，而是传递 hidden states。这就像人类交流时不需要复述所有历史，而是提炼关键信息。"

**内容**:
- **核心思想**: 在 latent space 进行通信
- **Memory Block**: 历史信息分块存储
- **Block Summary**: 每个 block 用向量表示语义

**📊 图示建议**:
```
[图：LatentMAS 架构图]

Agent A → [完整文本: "首先我们需要...然后...最后..."] → Agent B
         (1000 tokens)

LatentMAS:
Agent A → [Block 1] [Block 2] [Block 3] ... [Block N] → Agent B
         (每个 block 是一个向量，无需解码)

Memory Pool:
┌────────┬────────┬────────┬────────┬────────┐
│Block 1 │Block 2 │Block 3 │  ...   │Block N │
│Summary │Summary │Summary │        │Summary │
└────────┴────────┴────────┴────────┴────────┘
```

## Slide 6: 研究目标

**演讲要点**:
> "我们的目标不是追求极致的压缩，而是在保持准确率的前提下，尽可能减少计算量。具体来说..."

| 目标类型 | 指标 | 阈值 |
|----------|------|------|
| **主要目标** | 准确率下降 | < 5% |
| **次要目标** | Latent Reduction | 最大化 |

**关键洞察**:

- 在准确率可接受范围内找最优 trade-off
- 策略网络本身不能太大

## Slide 8: 策略网络架构

**演讲要点**:
> "我们选择 Cross-Attention 架构，因为它天然适合 query-key 匹配问题。Query 作为 Q，blocks 作为 K 和 V。"

**内容**:

**为什么选择 Cross-Attention?**
1. 自然的 query-key 匹配机制
2. 支持可变数量的 blocks
3. 全局信息聚合

**网络结构**:
```python
输入:
  - query_embed: [B, D]        # 当前问题的 embedding
  - block_summaries: [B, N, D] # N 个 blocks 的 summary

处理流程:
  1. Query Projection: Linear(D → D)
  2. Cross-Attention × 2 层
  3. Score Head: MLP → logits [B, N]

输出:
  - 每个 block 的选择得分
  - Top-K 采样得到选中的 blocks
```

**📊 图示建议**:
```
[图：Cross-Attention 层详细结构]

Query ──→ [Linear] ──→ Q
                        ↘
Blocks ──→ [Linear] ──→ K  → [Attention] → [FFN] → Output
                        ↗
Blocks ──→ [Linear] ──→ V
```

**设计决策表**:

| 参数 | 选择 | 理由 |
|------|------|------|
| hidden_dim | 2560 (与LLM一致) | 无需额外投影 |
| num_heads | 4 | 平衡表达力和效率 |
| num_layers | 2 | 实验验证足够 |
| 参数量 | ~62M | 相比 4B LLM 很轻量 |

## Slide 9: GRPO 训练算法

**演讲要点**:
> "传统 RL 算法如 PPO 需要额外的 value network，增加训练复杂度。GRPO 通过组内相对排名来估计 advantage，更简单高效。"

**内容**:

**为什么不用 PPO/REINFORCE?**
- PPO: 需要 value network，训练复杂
- REINFORCE: 方差大，训练不稳定
- DPO: 需要成对偏好数据

**GRPO 核心思想**:
- 对每个问题采样 G 个不同选择
- 计算各选择的奖励
- 用组内相对排名代替 baseline

**📊 图示建议**:
```
[图：GRPO 训练流程]

问题 Q ──→ 策略网络 ──→ 采样 G=4 个选择
                            │
         ┌──────────────────┼──────────────────┐
         ↓                  ↓                  ↓
    选择 1: [B3,B7]    选择 2: [B1,B5]    选择 3: [B2,B8] ...
         ↓                  ↓                  ↓
    执行推理            执行推理            执行推理
         ↓                  ↓                  ↓
    奖励 r1=0.8        奖励 r2=0.3        奖励 r3=0.6
         │                  │                  │
         └──────────────────┼──────────────────┘
                            ↓
                    组内标准化奖励
                    r' = (r - mean) / std
                            ↓
                    策略梯度更新
                    ∇θ = Σ r'i · ∇log π(ai)
```

## Slide 10: 奖励函数设计

**演讲要点**:
> "奖励函数的设计是最关键的部分，也是我们踩坑最多的地方。最终发现，组合 task 和 consistency 奖励，同时去掉代价惩罚，效果最好。"

**内容**:

**奖励函数公式**:
$$R = \alpha \cdot R_{task} + \beta \cdot R_{consistency} - \gamma \cdot R_{cost}$$

**各分量定义**:

| 分量 | 公式 | 含义 | 特点 |
|------|------|------|------|
| R_task | 1 if correct else 0 | 任务正确性 | 稀疏信号 |
| R_consistency | 1 if answer=baseline else 0 | 与全量读取一致 | 稠密信号 |
| R_cost | selected/total | 读取代价 | 鼓励少选 |

**📊 图示建议**:
```
[图：奖励分量示意]

R_task (稀疏):      ████████░░░░░░░░  (只有答对才有奖励)
R_consistency (稠密): ████████████░░░░  (与 baseline 一致就有奖励)
R_cost (惩罚):       ░░░░░░░░████████  (选得越多惩罚越大)

最终奖励 = α×task + β×consistency - γ×cost
```

**关键发现预告**:
- γ > 0 会破坏学习（后面实验证明）
- 最优配置: α=0.5, β=0.5, γ=0

---

## Slide 11: 训练与推理流程

**演讲要点**:
> "训练时我们用 GRPO 采样多个选择并更新策略。推理时直接用训练好的策略选择 top-k blocks，然后只用这些 blocks 进行 LLM 推理。"

**📊 图示建议**:
```
[图：训练 vs 推理对比]

训练阶段:
┌────────────────────────────────────────┐
│ for each question:                     │
│   1. 采样 G 个不同的 block 选择        │
│   2. 对每个选择执行推理，计算奖励      │
│   3. GRPO 更新策略网络参数             │
└────────────────────────────────────────┘

推理阶段:
┌────────────────────────────────────────┐
│ for each question:                     │
│   1. 策略网络计算每个 block 的得分     │
│   2. 选择得分最高的 k 个 blocks        │
│   3. 只用这 k 个 blocks 进行 LLM 推理  │
└────────────────────────────────────────┘
```

## Slide 14: V2/V3 实验 - 寻找方向

**演讲要点**:

**V3 配置与结果**:
```python

```
→ GSM8K: -7.6% ❌, ARC: **-4.8%** ✅

**演讲要点**:

## Slide 19: 奖励函数消融

**演讲要点**:
> "这是最重要的消融实验。单独使用 task 或 consistency 奖励效果都不好，组合使用效果最佳。这说明两种信号是互补的。"

**实验配置**: GSM8K, k=40

**结果**:

| 配置 | α | β | γ | Δ Accuracy |
|------|---|---|---|------------|
| task_only | 1.0 | 0.0 | 0.0 | **-9.4%** ❌ |
| consistency_only | 0.0 | 1.0 | 0.0 | **-6.4%** ❌ |
| **balanced** | **0.5** | **0.5** | **0.0** | **-2.8%** ✅ |

## Slide 20: 关于代价惩罚 γ 的深入分析

**演讲要点**:
> "为什么代价惩罚反而有害？因为它让优化目标变得矛盾。策略既要选对，又要少选，两个目标互相冲突。"

**📊 图示建议**:
```
[图：优化目标冲突示意]

γ > 0 时的优化目标:
┌─────────────────────────────────────┐
│        max R_task + R_consistency   │ ← 要选对
│        min R_cost                   │ ← 要少选
│                                     │
│        这两个目标是矛盾的！          │
└─────────────────────────────────────┘

结果: 策略学会"随便选"来降低 cost
     ↓
  准确率大幅下降

γ = 0 时:
┌─────────────────────────────────────┐
│        max R_task + R_consistency   │ ← 只需要选对
│                                     │
│        目标清晰，学习更有效          │
└─────────────────────────────────────┘
```

**关键洞察**:
- Latent reduction 由 top_k 参数控制
- 策略只需学习"选对"，不需要学习"少选"
- 分离这两个目标让训练更稳定
